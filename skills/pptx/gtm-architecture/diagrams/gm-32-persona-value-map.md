# GM-32: Persona Value Map - GTM Diagram Spec

Ref: Persona-based messaging frameworks | Stakeholder value mapping

## 1. Purpose and When to Use
Definition: A map showing differentiated value propositions by buyer and user personas.
Use when: multi-stakeholder selling, messaging alignment, executive summary design.
Questions answered: What does each persona gain and what proof matters to them?
Primary audience: PMM, sales, leadership communication.

## 2. Visual Layout Specification
Variant A (recommended): Rows as personas, columns as pain, value, KPI, and proof.
Variant B: Circular map with central product and persona satellites.
Variant C: Prioritized persona map with influence and urgency dimensions.

**Grid Proportions**:
- 5-column table: 8.5" wide at (0.5", 1.1")
- Column widths: Persona 1.5", Pain 1.7", Value 1.7", KPI 1.7", Proof 1.5"
- Persona row height: 72pt; header row: 36pt, `#44546A` fill
- Persona label: 10pt bold with role subtitle 8pt below
- Influence badge: 12pt circle next to persona name (colored by influence level)
- Primary decision persona row: `#00CCD7` left accent bar (3pt)

## 3. Color Semantics
Primary decision persona: #00CCD7
Secondary persona: #53E3EB
Low influence persona: #A5A7AA
Header bars: #44546A

## 4. Typography Hierarchy
Title 28pt, subtitle 14pt, persona labels 10pt bold, value statements 8.5-9pt, footnote 7pt.

## 5. Shape and Connector Vocabulary
Table or persona cards with optional influence connectors.
Use influence markers only when needed.

## 6. Annotation Rules
Persona label should include role and decision influence.
Each persona must include one measurable KPI and one proof preference.
Avoid generic values repeated across all personas.

## 7. Content Density Rules
Minimum: 3 personas.
Optimal: 4-6 personas.
Maximum: 8 personas then split by buying stage.

## 8. Anti-Patterns
Single generic value statement for all personas.
No distinction between economic buyer and operational user.
No KPI linkage.

## 9. Industry Reference Patterns
Enterprise deals require persona-specific outcomes and proof styles.
High-performing value maps include decision influence weighting.

## 10. Production QA Checklist
- [ ] Persona influence is defined
- [ ] Persona-specific value statements are distinct
- [ ] KPI and proof preference are included per persona
- [ ] Economic and technical buyers are both represented
- [ ] Language matches persona vocabulary
