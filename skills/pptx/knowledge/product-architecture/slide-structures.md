# Slide Structures — 产品架构设计

_Last updated: 2026-03-25 | Templates recorded: 2_

## Templates

### Template: TOGAF 业务架构层（10图全覆盖）

- **Source**: manual
- **Uses**: 0
- **Audience**: 管理层 / 业务团队 / 售前
- **Slide count**: 12
- **Added**: 2026-03-25
- **Last used**: —

**Structure**:
1. Cover — 业务架构主题 | 范围与目标
2. Business Goals — 战略目标与约束
3. Capability Map — 业务能力地图
4. Value Stream — 价值流
5. Process (L1/L2) — 业务流程
6. Org + Actor Interaction — 组织与角色协作
7. Service Decomposition — 业务服务分解
8. Function-Capability Mapping — 功能与能力映射
9. As-Is vs To-Be — 现状与目标对比
10. KPI Alignment — KPI 与目标对齐
11. RACI Matrix — 治理责任矩阵
12. Close — 结论与行动

**Notes**:
- 该模板必须优先使用 `business-architecture` skill。
- 图形表达优先业务语义，不得退化为技术拓扑图。
- 连接线仅用于关系表达（流向/依赖/责任），禁止装饰线。

### Template: TOGAF 业务架构层（精简汇报版）

- **Source**: manual
- **Uses**: 0
- **Audience**: 管理层
- **Slide count**: 8
- **Added**: 2026-03-25
- **Last used**: —

**Structure**:
1. Cover — 主题与一句话目标
2. Capability Map — 关键能力版图
3. Value Stream — 端到端价值链路
4. Process Focus — 核心流程与瓶颈
5. As-Is vs To-Be — 业务转型差距

**Notes**:

- 以“目标-能力-流程-价值-治理”主线组织。
