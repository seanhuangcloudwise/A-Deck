# Diagram Patterns — 图形表达模式库

_Last updated: 2026-03-25 | Patterns documented: 21_

本文档记录 PPT 内容表达的图形模式与逻辑，用于替代纯文字列表。每个模式包含：适用场景、设计规范、最佳实践。

---

## 1. 并列 (Parallels / Side-by-side)

**适用场景**：
- 多个选项/形态同级对比
- 3-5 个独立要素的平行介绍
- 产品线/部件的横向展示

**实现方式**：
- 等宽卡片框（通常 3-5 列）
- 每列用其他颜色背景区分
- 左侧彩条强调

**示例内容**：
```
┌─────────┬─────────┬─────────┐
│ Option1 │ Option2 │ Option3 │
│ 特性A   │ 特性A   │ 特性A   │
│ 特性B   │ 特性B   │ 特性B   │
└─────────┴─────────┴─────────┘
```

**应用场景**：功能对比、部件介绍、多渠道展示

---

## 2. 列表 (Lists / Sequential)

**适用场景**：
- 3-7 个逐项要点
- 有优先级但无时间顺序
- 检查清单、特性列表

**实现方式**：
- 编号或符号（✓, >, ●）
- 彩色左侧竖条或前缀图标
- 每项独占一行或分组放置

**示例**：
```
① 特性一 — 描述
② 特性二 — 描述  
③ 特性三 — 描述
```

**应用场景**：功能清单、优势列表、实施步骤

---

## 3. 总分 (Hierarchy / Top-down)

**适用场景**：
- 一个主要观点 + 多个支撑点
- 金字塔型逻辑
- 一个题目，多个分论点

**实现方式**：
- 顶部大框（主观点）
- 下方 2-3 级副框
- 使用颜色渐变或大小递减表示层级

**示例**：
```
    ┌─────────────────┐
    │   主要结论      │
    └────┬────┬────┬──┘
        ┌┴┐  ┌┴┐  ┌┴┐
        │1│  │2│  │3│
        └──┘  └──┘  └──┘
```

**应用场景**：商业价值、解决方案概览、战略思路

---

## 4. 时间轴 (Timeline)

**适用场景**：
- 项目计划、发展阶段
- 2025 Q2 → Q3 → Q4 这样的时序
- 里程碑式展示

**实现方式**：
- 水平线索（或竖向）
- 时间点标记
- 每个点关联事件/成果

**示例**：
```
Q1 ───● Q2 ───● Q3 ───● Q4
      事项1    事项2    事项3
```

**应用场景**：产品路线图、交付计划、成长里程碑

---

## 5. 关系 (Relationships / Network)

**适用场景**：
- 多个实体相互关联
- 生态图、协作关系
- 依存关系展示

**实现方式**：
- 节点+连接线
- 不同颜色表示不同类型关系
- 箭头或虚线表示流向/影响

**示例**：
```
    ┌─────┐
    │ A   │ ←─┿─→ B-C-D
    └─────┘
```

**应用场景**：组件关系、生态图、价值链

---

## 6. 循环 (Cycle / Loop)

**适用场景**：
- 迭代式流程、持续改进
- 部件间循环影响
- 反馈闭环

**实现方式**：
- 圆形/环形排列
- 箭头形成闭合循环
- 通常 3-5 个阶段

**示例**：
```
    ┌─────┐
    │  1  │ ↘
    └─────┘   ┌─────┐
              │  2  │ ↘
              └─────┘   ┌─────┐
                        │  3  │ ↘
                        └──┬──┘   ↘ 回到 1
```

**应用场景**：持续改进、用户旅程、售后循环

---

## 7. 流程 (Process / Flow)

**适用场景**：
- 线性多步骤工作流
- 数据/任务转换过程
- 从 input 到 output 的路径

**实现方式**：
- 左右或上下方向的方框链接
- 箭头表示流向
- 菱形表示决策点

**示例**：
```
输入 → 处理1 → 处理2 → 输出
       ↓
     决策
       ↓
     分支
```

**应用场景**：处理流程、业务工作流、交付过程

---

## 8. 组织架构 (Org structure / Hierarchy)

**适用场景**：
- 团队结构、汇报线
- 产品线体系
- 模块依赖关系

**实现方式**：
- 树形布局（自上而下或自左至右）
- 父子框用线连接
- 层级明确

**示例**：
```
        总体系
       ╱   ╲
      模块1 模块2
      ╱  ╲
  部件  部件
```

**应用场景**：产品体系、团队组织、系统架构

---

## 9. 金字塔 (Pyramid)

**适用场景**：
- 优先级递减
- 从基础到高端的递进
- 价值链从下到上

**实现方式**：
- 梯形分层（宽底、尖顶）
- 通常 3-4 层
- 颜色或填充表示层级

**示例**：
```
        ▲
       ╱ ╲
      ╱   ╲  顶层
     ╱─────╲
    ╱       ╲  中层
   ╱─────────╲
  ╱           ╲ 基础层
 ╱─────────────╱
```

**应用场景**：需求优先级、消费者阶层、能力层次

---

## 10. 矩阵 (Matrix / Quadrant)

**适用场景**：
- 2 维度对标（如重要性 vs 紧急性）
- 选项分类（如成本 vs 复杂度）
- 产品定位图

**实现方式**：
- 二维坐标系（横轴+纵轴）
- 4 或 9 个象限
- 数据点/框体放置在对应象限

**示例**：
```
 高   ┌──────┬──────┐
 │  │  Q2  │  Q1  │
 │  ├──────┼──────┤
 │  │  Q3  │  Q4  │
 低  └──────┴──────┘
    低      成本      高
```

**应用场景**：战略定位、市场分析、功能优先级排序

---

## 11. 对比 (Comparison / Before-After)

**适用场景**：
- 现状 vs 目标
- 竞品对比
- 升级前后对照

**实现方式**：
- 左右或上下两列
- 鲜明的视觉差异（颜色、图标）
- 中间分割线或箭头指向

**示例**：
```
现状 ─────→ 目标
❌ 缺陷  │  ✓ 改进
❌ 问题  │  ✓ 解决
```

**应用场景**：方案优势、升级价值、竞品对标

## TOGAF 业务架构层专项图形（Business Layer)

## 12. 业务能力地图 (Capability Map)

**适用场景**：
- 展示能力版图、能力空白、建设优先级

**实现方式**：
- 按业务域分组的矩阵块
- 可加成熟度标签（L1/L2/L3）

**应用场景**：战略规划、能力建设路线

---

## 13. 价值流图 (Value Stream)

**适用场景**：
- 展示端到端价值创造链路

**实现方式**：
- 分阶段串联：触发 -> 处理 -> 交付 -> 反馈
- 每阶段附输出价值与 KPI

**应用场景**：流程优化、价值提升汇报

---

## 14. 业务流程图 (Process L1/L2)

**适用场景**：
- 业务流程梳理、瓶颈识别

**实现方式**：
- 线性流程或泳道流程
- 决策点必须有业务条件说明

**应用场景**：流程重构、SOP 设计

---

## 15. 组织与角色协作图 (Org/Actor Interaction)

**适用场景**：
- 跨部门协作关系说明

**实现方式**：
- 角色节点 + 协作边
- 边标签标注协作类型（审批/交付/沟通）

**应用场景**：治理机制、协同机制梳理

---

## 16. 业务服务分解图 (Service Decomposition)

**适用场景**：
- 从顶层服务分解到子服务

**实现方式**：
- 树状或分层总分结构
- 保持服务命名一致性

**应用场景**：服务目录定义、服务边界澄清

---

## 17. 功能-能力映射图 (Function-Capability Mapping)

**适用场景**：
- 分析功能建设对能力提升的覆盖度

**实现方式**：
- 行列映射矩阵或双侧映射图
- 缺口项显式标记

**应用场景**：建设规划、投资优先级

---

## 18. As-Is / To-Be 对比图

**适用场景**：
- 业务转型现状与目标差距表达

**实现方式**：
- 左右对照块 + 差距项
- 差距项需给对应改进动作

**应用场景**：转型方案、立项论证

---

## 19. 业务场景旅程图 (Scenario Journey)

**适用场景**：
- 说明业务场景中的关键触点与痛点

**实现方式**：
- 时间轴/阶段带 + 触点 + 痛点 + 结果

**应用场景**：客户旅程、运营流程优化

---

## 20. KPI-目标对齐图 (KPI Alignment)

**适用场景**：
- 目标、指标、责任人一体化表达

**实现方式**：
- 目标 -> KPI -> 目标值 -> Owner 链式结构

**应用场景**：经营管理、绩效对齐

---

## 21. RACI 责任矩阵

**适用场景**：
- 治理职责边界明确

**实现方式**：
- 活动 x 角色矩阵
- 单元格标记 R/A/C/I

**应用场景**：项目治理、跨部门协作

---

## 使用指南

**选择图形模式的流程**：

1. **识别内容逻辑**：
   - 并列？关系？流程？
   
2. **匹配对应模式**：
   - 参考本表的"适用场景"

3. **应用到 PPT**：
   - 用对应 Diagram Pattern 替代纯文字

4. **持续沉淀**：
   - 发现新的应用场景时，更新本文档

**最佳实践**：
- 不要混用过多模式（每页 1-2 个主模式即可）
- 颜色与 Cloudwise 标准调色板保持一致
- 保留必要文字标签，但减少冗余描述
- 优先用图形传达是什么，用文字说明为什么
- 所有连接线都必须具备语义（流向/依赖/责任），禁止纯装饰性线条
