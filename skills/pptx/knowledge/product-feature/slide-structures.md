**Structure**:
1. Cover — 功能名称 | 核心价值一句话
2. Problem — 用户痛点（故事化或数据驱动）
3. Solution Overview — 功能全貌（截图 + 标注 或 图示）
4. How It Works — 操作流程（步骤横排：3–5 步）
5. Key Benefits — 核心优势（3 卡片 或 图标行）
6. Use Cases — 典型使用场景（2–3 个场景卡片）
7. Close — Getting Started / 下一步

**Notes**: 图片密度高；截图/UI mock 优先；步骤图用横向流程布局
