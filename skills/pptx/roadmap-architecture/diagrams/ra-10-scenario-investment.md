# RA-10: 场景化投资视图 — Scenario Investment View

_Ref: McKinsey Scenario Planning | Shell Scenario Methodology | Ansoff Growth Matrix | Real Options Theory | Sequoia "RIP Good Times" Framework_

---

## 1. Purpose & When to Use

**Definition**: 在两个或多个假设情景（保守/基准/激进）下，对比展示各举措/产品线的投资规模、预期回报和战略重心的差异，帮助管理层在不确定环境中做出"情景感知的投资决策"，并预设当外部条件变化时的投资切换逻辑。

**Use When**:
- 外部不确定性高时（经济波动/新技术出现/监管变化），年度预算规划无法锁死单一方案
- 向董事会/投资委员会展示"我们已经考虑了多种情景"的成熟风险意识
- 新市场进入或新产品立项前，论证不同市场假设下的投资回报
- M&A 尽调阶段，对比收购后不同整合深度的投资路径
- 长期战略规划（3-5 年），预设情景转换触发条件和决策规则

**Questions Answered**:
- 在保守/基准/激进三种情景下，我们的投资重心有何差异？
- 哪些举措在所有情景下都值得投资（必投项）？
- 哪些举措只在激进情景下才适合大量投入（赌注项）？
- 当外部信号显示需要切换情景时，我们有什么预设触发条件？
- 不同情景下，整体预算分配比例如何变化？

**Primary Audience**: CPO、CFO、CEO、董事会、战略规划团队

## 2. Visual Layout Specification

**Structure**: 场景列（3列）× 举措行（N行）的矩阵，每个单元格内用气泡大小/颜色表示投资规模/状态

### Variant A: 场景投资矩阵（适合预算决策）
- 列：保守（Conservative）/ 基准（Base）/ 激进（Aggressive）
- 行：举措/战略主题（5-10 条）
- 单元格内容：气泡大小=投资金额，颜色=投资类别，数字=FTE/K元
- 最右列：必投/条件投/暂缓 决策状态徽章
- Best for: 年度预算对齐、CPO/CFO 联合决策

### Variant B: 情景概率加权投资视图（适合概率思维受众）
- 三列情景各附概率权重（如 20% / 60% / 20%）
- 每行末尾计算"期望投资值"（EV = Σ 概率 × 各情景投资）
- 最终总预算 = EV 之和，并附"情景触发条件"表
- Best for: 财务/投资者受众、以概率思维驱动决策

### Variant C: 情景时间线（适合战略节奏规划）
- 横轴：时间（年度/季度）
- 三条不同颜色的投资曲线代表三种情景
- 在时间轴上标注"情景转换触发点"（Trigger Points）
- 当前时间点用竖线标注，说明"当前处于哪条曲线"
- Best for: 多年期战略规划、向 Sponsor 展示动态投资节奏

**Slide Proportions**:
- Title placeholder: 顶部 12%
- 矩阵区域: 占 75% 内容高度
- 表头（情景列名+概率）: 固定 40px
- 左侧举措名列: 18% 宽度
- 三个情景列: 各占剩余 82% 的三等份
- 底部决策说明条: 固定 48px（"情景切换触发条件"）

## 3. Color Semantics

情景颜色编码：

| 情景 | 颜色系 | 说明 |
|------|--------|------|
| 保守 Conservative | `#A5A7AA` 灰 | 低增长预期，防守性投资 |
| 基准 Base | `#00CCD7` 主青 | 最可能情景，基准预算 |
| 激进 Aggressive | `#44546A` 深蓝 | 高增长押注，进攻性投资 |

投资类别颜色（气泡填充）：

| 类别 | 颜色 | 说明 |
|------|------|------|
| 核心/必投 | `#00CCD7` | 所有情景均投 |
| 有条件投入 | `#53E3EB` | 基准或以上情景投 |
| 情景赌注 | `#44546A` | 仅激进情景投 |
| 暂缓/削减 | `#E5E5E5` | 保守情景削减或暂停 |

| 决策徽章 | 颜色 | 含义 |
|---------|------|------|
| 必投 | `#00BF87` 绿 | All-scenario commitment |
| 条件投 | `#FF9500` 橙 | Conditional on Base+ |

## 5. Shape & Connector Vocabulary

| Shape | Meaning |
|-------|---------|
| 圆形气泡（单元格内）| 该情景+该举措的投资规模（大小=金额） |
| 矩形单元格背景 | 情景×举措 分析格 |
| 圆角矩形徽章 | 决策状态标记（必投/条件投/暂缓） |
| 深色边框列 | 当前优选情景（高亮基准列） |
| 触发条件标旗 | 切换到不同情景的外部信号说明 |
| 汇总行（底部）| 各情景总投资对比柱状微图 |

**Connector Rule**: 场景列之间不使用连接箭头；情景切换逻辑通过底部"触发条件说明"栏以文本形式表达。

## 6. Annotation Rules

- **列总预算**: 每列底部标注该情景下的总投资额（"总预算: ¥XXX K/季度"）
- **必投标注**: 在举措行最右添加彩色决策徽章，三列一致的标 "必投 ✓"
- **情景触发条件**: 底部独立行标注 "保守情景触发: [如拿到融资<目标 70%]" / "激进情景触发: [如 Q2 MAU 增速 >30%]"
- **当前情景高亮**: 用粗边框（3pt `#2F2F2F`）高亮当前采用的情景列
- **脚注**: 数据版本 + 财务模型假设来源 + 更新责任人

## 7. Content Density Rules

| Mode | 举措行数 | 场景列数 | Max per Slide |
|------|---------|---------|---------|
| Minimum | 3 | 2 | — |
| Optimal | 5-8 | 3 | 8 rows |
| Maximum | 10 | 3 | → split by portfolio theme |

**Overflow Strategy**: 超过 10 行举措时，按战略主题分组（如"增长举措组" / "效率举措组"），每组一页，所有页共用同一列触发条件说明。

## 8. Anti-Patterns

1. **Single scenario masquerading（单情景伪装）**: 三列内容几乎相同，只有数字微调 — 失去情景规划的意义，不如直接用单一计划表。
2. **No triggers defined（无触发条件）**: 情景列有了，但没有说明"什么外部信号触发切换" — 情景规划的价值在于预设决策规则，否则只是静态假设。
3. **Missing current scenario marker（没有标注当前情景）**: 三列并排但未说明"当前采用哪个情景" — 受众不知道当前预算依据哪个假设。
4. **Too many scenarios（情景过多）**: 超过 4 种情景 — 每增加一种情景管理复杂度指数级上升，人类决策偏好 2-3 个对比选项，超过则需要权重合并。
5. **Scenario without probability（情景无概率权重）**: 三种情景并排但没有说明各自的发生可能性判断 — 管理层无法判断"最应该按哪个情景做资源准备"。

## 9. Industry Reference Patterns

**McKinsey Scenario Planning（战略情景规划）**:
麦肯锡的情景规划方法强调：情景不是预测，而是"结构性不同的未来"——真正有效的情景在核心假设上必须相互排斥且集体穷举（Mutually Exclusive, Collectively Exhaustive）。麦肯锡情景规划通常从"两个关键不确定性维度"生成 2×2 矩阵（而非线性的保守/基准/激进），每个象限命名为生动的场景名（如 "Digital Surge" / "Green Revival"），使管理层能直觉性理解每个情景的特征。RA-10 的线性三情景是简化版，适合 PowerPoint 展示；完整情景规划建议结合 2×2 Workshop 生成情景。

**Shell Scenario Methodology（壳牌情景方法论）**:
Shell 是情景规划最经典的企业实践者（Pierre Wack, 1970s），其方法论特别强调"预制触发条件（Pre-determined elements）+ 关键不确定性（Critical Uncertainties）"的区分。Shell 在每次情景发布时都制定"Early Warning Indicators"——一组可观测的外部信号，当这些信号出现时触发情景切换。RA-10 底部的"情景触发条件"栏直接从 Shell 方法论学习，将抽象情景转化为可操作的决策触发器。

**Real Options Theory（实物期权理论）**:
实物期权将金融期权理论应用于战略投资决策：将"小额先投"视为购买一个期权的权利（而非立即全额投入），在不确定性降低后再决策是否行权（加仓）。RA-10 的"必投"举措相当于直接买入底层资产；"条件投"举措相当于购买看涨期权（待基准情景确认后行权加仓）；"暂缓"举措相当于等待市场信号。这种框架帮助 CPO 向 CFO 解释"为什么我们在探索性举措上分批小额投资，而非一次性全额承诺"。

**Sequoia "Rip Good Times" & Black Swan Scenario**:
2008 年金融危机期间，Sequoia Capital 向被投企业发出著名的 "RIP Good Times" Memo，要求企业立即制定保守情景下的"生存预算（Survival Budget）"。这一案例完美展示了 RA-10 的核心价值：当外部触发条件出现（如资本市场收紧），企业应立刻从"激进情景"切换到"保守情景"，而不是仓促地临时砍预算。预制了保守情景预算的企业比没有预制的企业平均快 3 倍完成削减和调整。

## 10. Production QA Checklist

- [ ] 包含至少 2 种对比情景（推荐 3 种：保守/基准/激进）
- [ ] 每种情景有概率权重说明（合计 ≈ 100%）
- [ ] 当前采用的情景有粗边框高亮
- [ ] 底部有情景切换触发条件说明（每个情景至少 1 条）
- [ ] 每列底部有该情景的总预算/总 FTE 汇总
- [ ] 每行右侧有决策状态徽章（必投/条件投/暂缓）
- [ ] 举措行数 ≤ 10（超过则分组）
- [ ] 颜色图例说明：情景颜色 + 投资类别颜色
- [ ] 标题 / 副标题使用版式占位符（idx=0 / idx=1）
- [ ] Presenter 能用此图说明：当前按哪个情景运行、切换到另一情景的触发条件是什么
